# Product Requirements Document (PRD)
**Project Name:** AI Barber WhatsApp Bot  
**Version:** 1.0  
**Target Audience:** عملاء صالون الحلاقة (ذكور)  
**Tone of Voice:** لهجة مصرية ودودة، شبابية، ومرحة ("يا باشا"، "يا غالي"، "شياكة").  

---

## 1. نظرة عامة على المنتج (Product Overview)
روبوت محادثة (Chatbot) متكامل عبر تطبيق WhatsApp مخصص لصالونات الحلاقة. يهدف البوت إلى تحسين تجربة العملاء من خلال عرض قصات الشعر، توفير ميزة "التجربة الافتراضية" (Virtual Try-on) باستخدام الذكاء الاصطناعي بتكلفة منخفضة، وتقديم اقتراحات ذكية بناءً على شكل وجه العميل، ثم إتمام عملية الحجز المبدئي وإشعار الحلاق، مع اعتماد الدفع النقدي (كاش) في الصالون.

---

## 2. النطاق والميزات الأساسية (Core Features)

*   **القائمة التفاعلية والمستشار الذكي (Interactive Menu & Smart Stylist):**
    *   عرض قائمة بقصات الشعر المتاحة للاختيار المباشر.
    *   إدراج خيار صريح في القائمة: "سيب الطلعة دي عليا" للسماح للبوت باختيار القصة المناسبة بناءً على شكل وجه العميل.
*   **معالجة الصور بالذكاء الاصطناعي (AI Virtual Try-On):**
    *   استقبال صورة سيلفي للعميل.
    *   تحديد شكل الوجه (في حال اختيار المستشار الذكي).
    *   دمج قصة الشعر المختارة أو المقترحة مع صورة العميل وعرض النتيجة.
*   **التحقق الآلي من جودة الصورة (Image Validation):**
    *   التأكد من أن الصورة تحتوي على شخص واحد فقط وأن الوجه واضح قبل معالجتها.
*   **نظام إشعار الحجز (Booking Notification):**
    *   تأكيد الحجز بناءً على موافقة العميل.
    *   إرسال إشعار فوري وتلقائي للحلاق يحتوي على بيانات العميل والقصة المطلوبة.

## 3. مسار المستخدم وتجربة المحادثة (User Flow & Script)

| الخطوة | الإجراء من العميل | رد البوت (Script) | المنطق البرمجي (Logic) |
| :--- | :--- | :--- | :--- |
| **1. الترحيب** | إرسال رسالة البدء | "يا هلا بيك يا باشا في صالون [اسم الصالون]! ✂️ اختار القصة اللي تعجبك من المنيو، أو لو محتار سيب الطلعة دي عليا وأنا اختارلك اللي يليق عليك:" (عرض قائمة منسدلة/أزرار) | عرض WhatsApp Interactive List تحتوي على أسماء القصات + خيار "اختارلي القصة". |
| **2-أ. الاختيار المباشر** | اختيار قصة محددة | "اختيار ممتاز! عشان أوريك الشياكة، ابعتلي صورة سيلفي حلوة ليك لوحدك ووشك للكاميرا." | انتظار إدخال نوعه (Image). |
| **2-ب. طلب المساعدة** | اختيار "سيب الطلعة دي عليا" | "يا سيدي عينيا! ابعتلي سيلفي رايق ليك دلوقتي، والذكاء الاصطناعي هيحلل ملامحك ويختارلك أكتر قصة تظبط معاك 100% 📸😎" | توجيه لـ Flow تحليل الوجه، وانتظار (Image). |
| **3. التحقق من الصورة** | إرسال صورة سيئة/جماعية | "الصورة دي مش واضحة أوي يا غالي، أو فيها ناس تانية معاك. أستأذنك تاخد سيلفي رايق لوحدك وتكون الإضاءة فيه حلوة." | تكرار الطلب (Loop) حتى استلام صورة مطابقة. |
| **4. المعالجة (AI)** | إرسال صورة صحيحة | "استلمت يا ريس! ثواني بحلل تفاصيل الوش... بص يا سيدي، عملتلك بروفة سريعة، إيه رأيك في الشياكة دي؟" (إرفاق الصورة). | إرسال الـ API Request، استلام الصورة المدمجة وإرسالها للمستخدم. |
| **5. اتخاذ القرار** | رؤية النتيجة | (أزرار: "اعتمد واحجز" / "جرب قصة تانية") | توجيه حسب الزر المختار. |
| **6. تأكيد الحجز** | الضغط على "اعتمد واحجز" | "تم يا ريس! بعتنا رسالة للمعلم وهيكون في انتظارك. الدفع هيكون كاش في الصالون، تنورنا في أي وقت!" | Trigger webhook لإرسال إشعار لرقم الحلاق. |
| **7. معالجة الأخطاء** | إرسال نص غير مفهوم | "معلش يا غالي، مفهمتش قصدك أوي. بس لو حابب تحجز أو تشوف القصات، دوس على الزرار اللي تحت ده:" (زر القائمة) | Fallback message لإعادة توجيه المستخدم للنطاق الصحيح. |

## 4. المتطلبات التقنية (Technical Specifications)

*   **WhatsApp Business API:**
    *   استخدام مزود خدمة (BSP) مثل WATI، ManyChat، أو Chatfuel.
    *   الاعتماد على واجهات المستخدم التفاعلية (Interactive Messages).
*   **AI Integration (Low Cost Stack):**
    *   **Vision API:** استخدام **Google Gemini 3 Flash Preview** (مجاني عبر Google AI Studio) لتحليل شكل الوجه بخاصية الـ Vision، بحيث يكون الـ Prompt مخفياً: "Determine face shape, output only the shape name".
    *   **Image Swap/Inpainting API:** الربط مع Replicate API (نماذج FLUX Schnell المفتوحة) أو Hugging Face API (كخطة بديلة) لتنفيذ عملية تبديل الشعر (Hair Swap) بتكلفة تعتمد على الاستخدام (Pay-per-API call).
*   **Notification System:**
    *   إعداد Webhook أو أتمتة بسيطة (مثل Make أو Zapier) لإرسال رسالة نصية أو واتساب لرقم هاتف الحلاق عند تأكيد الحجز.

## 5. المتطلبات غير الوظيفية والأمان (Non-Functional Requirements)

*   **السرعة (Performance):** يجب أن تتم عملية تحليل الصورة وتوليد النتيجة النهائية في غضون 15-25 ثانية كحد أقصى.
*   **الخصوصية (Privacy):** عدم تخزين صور العملاء في قواعد البيانات. يتم معالجة الصورة في الذاكرة المؤقتة للـ API وحذفها فور إرسال النتيجة. يجب إضافة إشعار صغير للعميل: *"متقلقش، صورتك في أمان وبتتمسح فوراً."*
*   **تكلفة التشغيل (Cost Efficiency):** الاستغناء عن النماذج المدفوعة شهرياً والاعتماد على الـ API المرن للحفاظ على تكلفة تشغيلية منخفضة لصالون الحلاقة.

## 6. مقاييس النجاح (Success Metrics)

1.  نسبة العملاء الذين يكملون مسار المحادثة حتى تأكيد الحجز (Conversion Rate).
2.  نسبة استخدام ميزة "سيب الطلعة دي عليا" مقارنة بالاختيار اليدوي.
3.  تقليل نسبة الأخطاء في توليد الصور (Failed API calls).
